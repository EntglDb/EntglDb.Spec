The Vision

The Database for the
Decentralized Web

EntglDb (Entangled Database) is built to empower applications that must run everywhere: from high-end servers to edge devices, without relying on a central authority.

Offline-First

Applications continue to work seamlessly without internet connection. Data syncs automatically when online.

Peer-to-Peer

Direct device-to-device communication (Mesh Networking) bypasses the need for central cloud servers.

03

Architecture

Built for Resilience

A modular, layered architecture designed for portability and performance.

Core Layer

Handles Validations, CRDTs (Conflict-free Replicated Data Types), and Vector Clocks.

Network Layer

Abstracts transport protocols (TCP, WebSocket, Bluetooth) for agnostic P2P communication.

Persistence Layer

Pluggable storage engines. Currently supports SQLite, with planned support for LevelDB and others.

API Layer

Simple, idiomatic APIs for .NET, Kotlin, Node.js, and more.

04

Data Model

Document Oriented

Flexible schema-less JSON documents organized in Collections.

{
  "_id": "doc-uuid-1234",
  "_version": "1-nodeA",
  "type": "sensor_reading",
  "data": { "temp": 24.5, "unit": "C" },
  "timestamp": 1709230000
}

Every document carries its own version history (Vector Clock), enabling precise causal ordering across the distributed system.

05

Synchronization

Eventual Consistency

EntglDb guarantees that all nodes will eventually reach the same state.

  • Hybrid Logical Clocks (HLC)
    Combines physical time with logical counters for robust ordering.
  • Anti-Entropy
    Efficiently exchanges missing data using Merkle Trees or Delta lists.
  • Gossip Protocol
    Rapid information dissemination across the cluster.
06

Security

Trust No One

Security is not an afterthought; it's embedded in the protocol.

E2E Encryption

All communication channels are encrypted using TLS 1.3 or equivalent secure tunnels.

Authentication

Nodes authenticate via cryptographic handshakes (Ed25519 signatures) or OAuth2 tokens.
